# 코드 작성 스킬

coder 에이전트에 위임하여 실행한다.

$ARGUMENTS 에 구현할 기능 설명이나 계획 참조가 들어옵니다.

## 자율 운영 원칙
- 묻지 않고 실행한다 -- 판단이 필요하면 직접 판단하고 결과를 보고
- 환경 문제(패키지 누락, import 에러)는 자동 설치 후 진행
- 에러 발생 시 3회까지 자동 재시도 후 보고

## 수행 절차

1. **컨텍스트 확인**
   - 이전 대화에서 /plan 결과가 있으면 해당 계획을 따름
   - 없으면 관련 코드를 빠르게 탐색하여 패턴 파악

2. **코드 작성 규칙**
   - Python 백엔드: FastAPI 라우터 패턴, Pydantic 스키마, 서비스 레이어 분리
   - React 프론트엔드: 함수형 컴포넌트, hooks, props destructuring
   - 기존 코드 스타일을 최우선으로 따름
   - 변경하지 않은 코드에 주석/독스트링/타입 힌트를 추가하지 않는다. 변경한 코드는 coder 에이전트 규칙을 따른다.

3. **작성 순서**
   - 백엔드 모델/스키마 먼저
   - 서비스 로직
   - API 라우터
   - 프론트엔드 컴포넌트
   - 각 단계마다 TaskCreate/TaskUpdate로 진행 추적

4. **완료 후**
   - 변경 파일 목록 요약
   - `/test-run` 으로 테스트 실행 권장
   - git diff 확인 가능하도록 안내
