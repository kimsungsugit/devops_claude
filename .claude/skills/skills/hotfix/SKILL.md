# 긴급 버그 수정 워크플로우

$ARGUMENTS 에 버그 설명, 에러 메시지, 또는 이슈 번호가 들어옵니다.

## 자율 운영 원칙
- **전 과정 자동 실행** — 분석→수정→테스트→커밋을 묻지 않고 완료
- 최소 변경 원칙 — 버그 원인만 수정, 리팩토링 금지
- 실패 시 3회 자동 재시도 후 보고

---

### STEP 1: 버그 분석
- 에러 메시지나 증상으로 `Grep` 검색
- 원인 파일과 라인 특정
- 보고: "원인: [파일:라인] — [설명]"

### STEP 2: 수정
- 버그 원인만 정확히 수정
- 변경 범위 최소화

### STEP 3: 테스트
- 관련 테스트 + 전체 단위 테스트 실행
- 실패 시 자동 수정 → 재실행 (최대 3회)

### STEP 4: 자동 커밋
- `fix: [간결한 설명]` 형식으로 **즉시 커밋**
- 결과 보고:
  ```
  ## Hotfix 완료
  버그: [증상]
  원인: [파일:라인] — [근본 원인]
  수정: [변경 내용]
  테스트: PASS
  커밋: abc1234 "fix: ..."
  ```
