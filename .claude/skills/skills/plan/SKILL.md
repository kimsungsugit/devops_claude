# 기획 및 구현 계획 스킬

planner 에이전트에 위임하여 실행한다. 이 스킬은 planner의 진입점이다.

$ARGUMENTS 에 기획할 기능이나 작업이 들어옵니다.

## 수행 절차

1. **현황 분석**
   - planner 에이전트가 관련 코드를 Glob/Grep/Read로 탐색
   - 기존 구현 패턴과 아키텍처 파악
   - 영향받는 파일 목록 정리

2. **요구사항 정리**
   - 사용자 요청을 구체적 요구사항으로 분해
   - 기능적/비기능적 요구사항 구분
   - 선행 조건 및 의존성 파악
   - ISO 26262 안전 요구사항 해당 여부 판단 (ASIL 등급 확인)

3. **구현 계획 작성**
   - 변경할 파일 목록과 각 파일의 변경 내용
   - 새로 생성할 파일 (필요시)
   - API 변경사항 (엔드포인트, 스키마)
   - 프론트엔드 변경사항 (컴포넌트, 라우팅)
   - 요구사항-구현 추적성 매핑 (어떤 요구사항이 어떤 파일/함수에 반영되는지)

4. **테스트 계획**
   - 단위 테스트 추가/수정 항목
   - 통합 테스트 시나리오
   - 수동 검증 체크리스트

5. **리스크 및 대안**
   - 잠재적 이슈
   - 대안 접근법
   - 롤백 전략

## 출력 형식

```markdown
## 구현 계획: [기능명]

### 변경 파일
| 파일 | 변경 유형 | 설명 | 안전 영향도 |
|------|----------|------|------------|
| ... | 수정/신규/삭제 | ... | 없음/ASIL A~D |

### 추적성
| 요구사항 ID | 구현 파일 | 검증 방법 |
|------------|----------|----------|
| REQ-xxx | file.c | UT / IT |

### 구현 순서
1. ...
2. ...

### 테스트 계획
- [ ] ...

### 리스크
- ...
```
